## 已记录的策略

### 💡 S-001 策略面板在 Web Dashboard 上显示
- **提出时间**: 2026-04-04
- **影响模块**: Phase 5 — 前端 Dashboard
- **状态**: 🟢 已采纳
- **描述**: STRATEGY.md 中的想法和策略记录，应该在 Web Dashboard 上有专属面板展示，让决策过程透明可见。
- **对比当前方案**: 当前 STRATEGY.md 只是本地 Markdown 文件，Dashboard 没有对应页面。
- **结论/备注**: 新增 GET /api/strategy 端点，从文件实时解析并渲染到 StrategyPanel.vue。

---

### 💡 S-002 自动查找免费 API Key 注册入口
- **提出时间**: 2026-04-04
- **影响模块**: Phase 0 — 配置 / Phase 1 — 数据采集
- **状态**: 🟢 已采纳
- **描述**: 自动搜索 MetalpriceAPI / GoldAPI / FRED / NewsAPI / ExchangeRate-API 的免费注册地址，减少手动查找成本。
- **对比当前方案**: .env 中 key 留空，需用户自行注册。
- **结论/备注**: 已整理免费层信息，见 log.md API Keys 章节。

---

### 💡 S-003 支持自定义 AI 模型端点（本地LLM / 兼容OpenAI接口）
- **提出时间**: 2026-04-05
- **影响模块**: Phase 3 — AI 分析层 / src/processors/ai/claude-client.ts
- **状态**: 🟢 已采纳
- **描述**: 除 Anthropic Claude API 外，支持用户指定自定义模型端点（IP地址）、API Key 和模型名称。兼容 OpenAI Chat Completions 格式（Ollama / LM Studio / vLLM / 本地部署模型均适用）。
- **对比当前方案**: 当前硬绑定 Anthropic SDK，无法使用本地模型或第三方兼容接口。
- **结论/备注**: 新增 AI_CUSTOM_BASE_URL / AI_CUSTOM_API_KEY / AI_CUSTOM_MODEL 三个环境变量。若 AI_CUSTOM_BASE_URL 有值则优先走自定义端点（axios调用），否则回退至 Anthropic SDK。

---

### 💡 S-004 黄金历史数据展示 + 用户想法 AI 分析工坊
- **提出时间**: 2026-04-05
- **影响模块**: Phase 1 采集层 / Phase 3 AI层 / Phase 4 API层 / Phase 5 前端
- **状态**: 🟢 已采纳
- **描述**: 三个子功能：①多时间框架历史K线（1M/3M/1Y/5Y）；②「想法工坊」——用户输入交易想法，AI结合当前行情+宏观数据+历史想法给出结构化分析；③想法持续积累，AI分析时参考过往想法+实际市场走向，形成「思维复利」。
- **对比当前方案**: 当前只有7天分钟级数据，无用户输入通道，AI只分析新闻。
- **结论/备注**: 新增 user_ideas 表 + GET/POST /api/ideas + HistoricalChart + IdeaWorkshop 两个组件。AI分析引用过去最近5条想法作为上下文，实现连续优化。

### 💡 S-005 核心目标重对焦：积存金短线交易赚钱
- **提出时间**: 2026-04-05
- **影响模块**: 全栈 — 新增交易信号引擎 + 交易日志 + 盈亏追踪
- **状态**: 🟢 已采纳
- **描述**: CEO评审发现产品目标错位——系统是情报展示台，但用户核心目标是积存金短线赚钱。需补齐三大缺口：①规则型交易信号引擎（STRONG_BUY/BUY/HOLD/SELL/STRONG_SELL），②积存金交易日志（买入/卖出记录+手续费），③实时盈亏追踪（均成本/浮动盈亏/已实现盈亏）。
- **对比当前方案**: 当前系统看数据不出信号，没有交易记录，不知道赚没赚钱。
- **结论/备注**: 新增 signal-engine.ts（规则驱动，RSI+MACD+布林带+SGE溢价+新闻情绪合成评分）、trade_log表、signals表、SignalPanel.vue、TradeLog.vue。信号置于Dashboard最顶部。
